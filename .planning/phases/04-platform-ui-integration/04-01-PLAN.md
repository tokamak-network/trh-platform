version: "3.8"

services:
  crosstrade-dapp:
    image: tokamaknetwork/cross-trade-app:latest

Key decisions:
- version "3.8" matches existing resources/docker-compose.yml (PLT-01)
- Port 3004:3000 maps external 3004 to container's internal 3000 (standard Next.js port)
- env_file references config/.env.crosstrade RELATIVE to compose file location
- No `networks:` block needed: dApp communicates with L2 via host network URLs, not internal Docker DNS
- No `depends_on:` needed: dApp is standalone after env file is written

NOTE: This file in `resources/` serves as the canonical template. It is also bundled into the Electron app package. The Backend will write this file to `stack.DeploymentPath/` at runtime (see Task 2) so that the relative `env_file: ./config/.env.crosstrade` path resolves correctly alongside the already-written `.env.crosstrade`.

In `deployment.go`, find the CrossTrade success block. After the `.env.crosstrade` write block (after the `if envErr` block, around line 348) and BEFORE the `stack.Metadata.CrossTradeUrl` line, add the BE-08 dApp compose file write and container start:

**Path rationale:** The Backend runs inside a Docker container where `stack.DeploymentPath` resolves to `/app/storage/deployments/{stack}/{network}/{id}`. The `.env.crosstrade` is already written to `stack.DeploymentPath/config/.env.crosstrade` (line 344). The compose file's `env_file: ./config/.env.crosstrade` is a RELATIVE path, so the compose file must live at the same level as the `config/` directory — i.e., `stack.DeploymentPath/docker-compose.crosstrade.yml`. This matches the existing pattern where deployment artifacts live in `stack.DeploymentPath`.

```go
// BE-08: Write docker-compose.crosstrade.yml and start dApp container (non-fatal).
// The compose file must be at stack.DeploymentPath so that its relative
// env_file (./config/.env.crosstrade) resolves alongside the env file
// already written above.
crossTradeComposePath := filepath.Join(stack.DeploymentPath, "docker-compose.crosstrade.yml")
crossTradeComposeContent := `version: "3.8"

services:
  crosstrade-dapp:
    image: tokamaknetwork/cross-trade-app:latest
    container_name: trh-crosstrade-dapp
    ports:
      - "3004:3000"
    env_file:
      - ./config/.env.crosstrade
    restart: unless-stopped
`
if writeErr := os.WriteFile(crossTradeComposePath, []byte(crossTradeComposeContent), 0644); writeErr != nil {
    logger.Warn("failed to write docker-compose.crosstrade.yml (non-fatal)",
        zap.String("stackId", stackId.String()), zap.Error(writeErr))
} else {
    startOut, startErr := exec.CommandContext(ctx,
        "docker", "compose", "-f", crossTradeComposePath, "up", "-d",
    ).CombinedOutput()
    if startErr != nil {
        logger.Warn("failed to start CrossTrade dApp container (non-fatal)",
            zap.String("stackId", stackId.String()),
            zap.Error(startErr),
            zap.String("output", string(startOut)))
    } else {
        logger.Info("CrossTrade dApp container started",
            zap.String("stackId", stackId.String()),
            zap.String("composePath", crossTradeComposePath))
    }
}
```

Verify that `os` and `os/exec` are already imported in deployment.go. Both should already be present since `filepath.Join` and `exec.CommandContext` are used elsewhere. If `os.WriteFile` is not available (Go < 1.16), use `os.MkdirAll` + `os.WriteFile` — but Backend uses Go 1.22+ so `os.WriteFile` is fine.

PLT-02 enforcement: This code only runs inside the `if enabled, ok := def.Modules["crossTrade"]; ok && enabled` block, which is only true for DeFi/Full presets. Gaming presets have crossTrade=false, so this code is never reached for Gaming.

Non-fatal pattern: Matches existing patterns in deployment.go where .env.crosstrade write failure and metadata update failure are both logged as Warn but don't block deployment.
