    **파일 2: `../trh-sdk/pkg/stacks/thanos/cross_trade_local_bytecodes.go`**
    **패키지:** `package thanos`

    crossTrade repo의 forge build output에서 4개 컨트랙트의 creation bytecode를 hex string 상수로 추출하여 정의한다.
    이렇게 분리하면 Input 구조체가 PRD v2.1 원형을 유지하면서도, 긴 bytecode hex string이 메인 로직 파일을 오염시키지 않는다.

    ```go
    package thanos

    // CrossTrade L2 contract creation bytecodes.
    // Extracted from crossTrade repo forge build output.
    // Source: crossTrade/out/{ContractName}.sol/{ContractName}.json "bytecode.object" field

    var (
        // L2CrossTrade implementation creation bytecode
        L2CrossTradeBytecode = mustDecodeHex("0x...")

        // L2CrossTradeProxy creation bytecode
        L2CrossTradeProxyBytecode = mustDecodeHex("0x...")

        // L2toL2CrossTradeL2 implementation creation bytecode
        L2toL2CrossTradeL2Bytecode = mustDecodeHex("0x...")

        // L2toL2CrossTradeProxy creation bytecode
        L2toL2CrossTradeProxyBytecode = mustDecodeHex("0x...")
    )

    func mustDecodeHex(hexStr string) []byte {
        b, err := hex.DecodeString(strings.TrimPrefix(hexStr, "0x"))
        if err != nil {
            panic(fmt.Sprintf("invalid bytecode hex: %v", err))
        }
        return b
    }
    ```

    **bytecode 추출 방법:** crossTrade repo에서 `forge build` 후, 각 JSON artifact의 `.bytecode.object` 필드를 추출.
    추출 대상:
    - `out/L2CrossTrade.sol/L2CrossTrade.json` -> `L2CrossTradeBytecode`
    - `out/L2CrossTradeProxy.sol/L2CrossTradeProxy.json` -> `L2CrossTradeProxyBytecode`
    - `out/L2toL2CrossTradeL2.sol/L2toL2CrossTradeL2.json` -> `L2toL2CrossTradeL2Bytecode`
    - `out/L2toL2CrossTradeProxy.sol/L2toL2CrossTradeProxy.json` -> `L2toL2CrossTradeProxyBytecode`

    **import 블록:** `encoding/hex`, `fmt`, `strings`
