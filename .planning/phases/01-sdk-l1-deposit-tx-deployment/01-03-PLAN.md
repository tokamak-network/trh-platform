    deployL2CrossTradePair 함수가 두 쌍 모두를 처리할 수 있도록 registerToken 로직을 범용화한다.

    **접근법:** registerToken 호출을 콜백 함수로 추출한다.

    ```go
    type registerTokenFunc func(proxyAddr common.Address, token TokenPair) ([]byte, error)
    ```

    deployL2CrossTradePair에 `registerTokenFn registerTokenFunc` 파라미터를 추가한다. Step 7에서 이 함수를 호출하여 calldata를 생성한다.

    **L2CrossTrade용 registerToken calldata 생성:**
    ```go
    // 3파라미터: registerToken(address l1Token, address l2Token, uint256 l1ChainId)
    l2CrossTradeRegisterToken := func(proxyAddr common.Address, token TokenPair) ([]byte, error) {
        return implABI.Pack("registerToken",
            common.HexToAddress(token.L1Token),
            common.HexToAddress(token.L2Token),
            l1ChainID,
        )
    }
    ```

    **L2toL2CrossTradeL2용 registerToken calldata 생성:**
    ```go
    // 6파라미터: registerToken(address l1token, address l2SourceToken, address l2DestinationToken, uint256 l1ChainId, uint256 l2SourceChainId, uint256 l2DestinationChainId)
    l2toL2RegisterToken := func(proxyAddr common.Address, token TokenPair) ([]byte, error) {
        return l2toL2ImplABI.Pack("registerToken",
            common.HexToAddress(token.L1Token),   // l1token
            common.HexToAddress(token.L2Token),    // l2SourceToken (같은 L2에서의 토큰)
            common.HexToAddress(token.L2Token),    // l2DestinationToken (동일 — single L2 시나리오)
            l1ChainID,                             // l1ChainId
            l2ChainID,                             // l2SourceChainId (현재 L2)
            l2ChainID,                             // l2DestinationChainId (동일 L2 — Phase 1)
        )
    }
    ```

    **deployL2CrossTradePair 함수 시그니처 변경:**
    `registerTokenFn registerTokenFunc` 파라미터를 마지막에 추가하고, Step 7의 기존 직접 Pack 호출을 `registerTokenFn(proxyAddr, token)` 호출로 교체.

    **Note:** L2toL2의 l2SourceToken과 l2DestinationToken이 동일한 이유: Phase 1은 단일 L2 배포. 다중 L2 간 크로스트레이드는 v2 범위.
