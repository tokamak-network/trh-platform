**ERC-4337 AA error code map** (for diagnostic logging):
```
AA93 -> invalid paymasterAndData (length < 52)
AA31 -> paymaster deposit too low
AA33 -> _validatePaymasterUserOp reverted
AA13 -> initCode failed or OOG
AA21 -> didn't pay prefund
AA25 -> nonce error
```

**Inline helper functions (ethers v6, do NOT import from SDK):**

1. `buildPaymasterAndData(tokenAddr)` -- 72-byte paymasterAndData:
   - ethers.concat([MULTI_TOKEN_PAYMASTER, zeroPadValue(toBeHex(150000n),16), zeroPadValue(toBeHex(50000n),16), tokenAddr])
   - Assert result is 72 bytes

2. `packUint128x2(high, low)` -- Pack two uint128 into bytes32:
   - (high << 128n) | low -> zeroPadValue(toBeHex(packed), 32)

3. `buildUserOpHash(userOp, chainId)` -- ERC-4337 userOpHash:
   - ABI encode [sender, nonce, keccak256(initCode), keccak256(callData), accountGasLimits, preVerificationGas, gasFees, keccak256(paymasterAndData)]
   - keccak256 of that, then encode with [innerHash, ENTRYPOINT_V08, chainId], keccak256 again

4. `signUserOpRaw(wallet, hash)` -- Raw ECDSA (NO EIP-191 prefix):
   - wallet.signingKey.sign(ethers.getBytes(hash))
   - ethers.Signature.from(sig).serialized

5. `sendUserOp(bundlerUrl, userOp)` -- JSON-RPC eth_sendUserOperation:
   - POST fetch with params: [serializedUserOp, ENTRYPOINT_V08]
   - Parse error: extract AA error code from message, log diagnostic
   - Return userOpHash

6. `waitForReceipt(bundlerUrl, hash, timeoutMs=60000)` -- Poll eth_getUserOperationReceipt:
   - 3-second interval, max 20 attempts
   - Return receipt or throw timeout

7. `pollUntil(fn, label, timeoutMs, intervalMs)` -- Reuse from bridge-tx.live.spec.ts pattern

**PackedUserOp type** (ERC-4337 v0.8 packed format):
```typescript
interface PackedUserOp {
  sender: string;
  nonce: string;
  initCode: string;
  callData: string;
  accountGasLimits: string;  // packUint128x2(verificationGasLimit, callGasLimit)
  preVerificationGas: string;
  gasFees: string;           // packUint128x2(maxPriorityFeePerGas, maxFeePerGas)
  paymasterAndData: string;
  signature: string;
}
```

**Test structure -- `test.describe('AA Paymaster Smoke Test')`:**

Test 1: `MultiTokenPaymaster bytecode injected into genesis`
- eth_getCode on proxy (0x...0067) -- expect not '0x', expect length > 10 (ERC-1967 proxy)
- eth_getCode on impl (0xc0D3...0067) -- expect length > 200 (full MultiTokenPaymaster bytecode)

Test 2: `EIP-7702 delegation active on admin EOA`
- Derive admin address from ADMIN_KEY
- eth_getCode on admin address -- expect starts with '0xef0100' (EIP-7702 delegation designator)
- If not set, test.skip() with diagnostic message

Test 3: `USDC balance and paymaster pre-conditions met`
- Check admin USDC balance via balanceOf(adminAddr) on USDC_ADDRESS -- expect > 0
- Check EntryPoint deposit for paymaster: call EntryPoint.balanceOf(MULTI_TOKEN_PAYMASTER) -- expect > 0
- Log balances for diagnostics

Test 4: `UserOp with USDC fee executes via Alto bundler`
- This is the main smoke test. In beforeAll or within the test:
  a. Get admin nonce from EntryPoint: `getNonce(adminAddr, 0)` via eth_call
  b. Get current gas prices from L2 provider (getFeeData)
  c. Build a minimal UserOp -- callData is a simple ETH transfer to self (0 value) via Simple7702Account.execute(target, value, data):
     - ABI encode: `execute(address,uint256,bytes)` with (adminAddr, 0, '0x')
  d. Set accountGasLimits = packUint128x2(200000n, 200000n)
  e. Set preVerificationGas = toBeHex(100000n)
  f. Set gasFees = packUint128x2(maxPriorityFee, maxFeePerGas) from provider feeData
  g. Set paymasterAndData = buildPaymasterAndData(USDC_ADDRESS) -- exactly 72 bytes
  h. Build userOpHash, sign with signUserOpRaw
  i. Send via sendUserOp to http://localhost:4337
  j. Wait for receipt via waitForReceipt
  k. Assert receipt exists and receipt.success === true (or receipt.receipt.status === '0x1')
- On bundler error: parse AA error code, log which layer failed (genesis/bundler/contract), fail with descriptive message
