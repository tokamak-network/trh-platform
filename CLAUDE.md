## Project Overview

**TRH Platform** — Electron 데스크톱 앱. Docker Compose로 PostgreSQL + Node.js backend + Next.js frontend를 로컬에서 실행하고, Terraform으로 AWS EC2에 배포한다.

- Frontend: http://localhost:3000
- Backend API: http://localhost:8000
- PostgreSQL: localhost:5432 (postgres/postgres)
- Default admin: admin@gmail.com / admin

## Common Commands

**Service Management**:
```bash
make setup       # Initial setup: create configs + docker compose up + backend init
make up          # Start all services
make down        # Stop containers (with confirmation)
make clean       # Stop + remove volumes
make status      # Container status
make logs        # Stream logs
make update      # Pull latest images + restart
make config      # Interactive env configuration
```

**EC2 Deployment**:
```bash
make ec2-setup      # One-time: AWS credentials + SSH keys → ec2/.env
make ec2-deploy     # Full deploy: Terraform + remote provisioning
make ec2-update     # SSH in → git pull → docker pull → restart
make ec2-status     # Terraform state + instance info
make ec2-destroy    # Terminate all resources
```

## Configuration Files

| File | Purpose |
|------|---------|
| `config/.env.docker` | Docker image overrides (empty by default; images use `:latest`) |
| `config/.env.backend` | PostgreSQL, JWT, default admin |
| `config/.env.frontend` | `NEXT_PUBLIC_API_BASE_URL` |
| `ec2/.env` | Terraform variables (EC2 deploy only; generated by `make ec2-setup`) |

Templates in `config/*.template`. Run `make config` or `make setup` to generate.

## Critical Patterns

**EC2 Provisioning Flow**: `make ec2-setup` (one-time) → `make ec2-deploy` (Terraform + cloud-init + install.sh + make setup) → `make ec2-update` (incremental updates)

**Terraform state** (`ec2/terraform.tfstate`) is committed to git.

## Troubleshooting

- Services won't start → `make logs`, ensure Docker is running
- DB connection fails → check `config/.env.backend` PostgreSQL credentials
- Frontend can't reach backend → check `NEXT_PUBLIC_API_BASE_URL` in `config/.env.frontend`
- EC2 timeout → `make ec2-status` for instance health
- Terraform state corrupted → `make ec2-clean` + retry (destroys existing resources)
- Credentials mismatch on EC2 → `make ec2-setup` to reconfigure

## Projects

### TRH Preset Deployment Test Harness

TRH 플랫폼의 4가지 Preset(General, DeFi, Gaming, Full) 배포 흐름을 mock 기반으로 검증하는 테스트 suite.

- **Stack**: TypeScript/Vitest (unit/integration), Playwright (E2E)
- **Mock boundary**: 모든 외부 의존성(L1/L2 RPC, Docker, AWS)은 mock/stub
- **Location**: 모든 테스트 코드는 `tests/` 디렉토리

### CrossTrade Integration

DeFi/Full Preset 선택 시 CrossTrade 프로토콜 자동 통합. L1 Deposit Tx 기반 L2 컨트랙트 배포 + L1 setChainInfo 등록 + dApp 컨테이너 배포까지 자동화.

**Constraints**:
- 기존 AWS CrossTrade 코드 수정 금지 (새 함수/파일로 병존)
- Backend→SDK 호출은 Go 패키지 import (HTTP/exec 아님)
- L1 Deposit Tx만 사용 (Genesis Predeploy 금지)
- 로컬(Docker Compose) 전용
