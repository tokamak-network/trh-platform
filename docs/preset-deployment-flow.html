서비스 설치
Platform UI — 3단계 Preset Wizard
Step 1

프리셋 선택

  • General / DeFi / Gaming / Full
  • 모듈 구성 미리보기
  • 예상 배포 시간 안내
Step 2

기본 정보 입력

  • Chain Name · Network (testnet/mainnet)
  • Fee Token (TON / ETH / USDT / USDC)
  • L1 RPC URL · Beacon URL
  • Seed Phrase (BIP44 키 파생)
  • AWS Credentials (Helm 배포용)
Step 3

검토 & 펀딩 확인

  • ConfigReview — 설정 최종 확인
  • FundingStatus — L1 지갑 잔액 실시간
  • sequencer / batcher / proposer 주소 표시
  • 최소 잔액 충족 시 배포 버튼 활성화

배포 순서 — Backend → SDK 상세 흐름
Platform UI POST /api/v1/stacks/thanos/preset-deploy
{ presetId, chainName, network, feeToken, seedPhrase, l1RpcUrl, beaconUrl, awsCredentials, overrides }
Backend BIP44 키 파생 (seed phrase → 4개 계정)
admin · sequencer · batcher · proposer — 각각 m/44'/60'/0'/0/{0-3} 경로
Backend L1 펀딩 상태 확인
sequencer / batcher / proposer 잔액 폴링 — testnet: 0.5 ETH · mainnet: 2 ETH 이상 필요
Backend trh-sdk 호출
thanos deploy --preset <id> --fee-token <token> --network <net> --chain-name <name> ...
trh-sdk Deploy Config 초기화 (프리셋별 오버라이드)
batchSubmissionFrequency · outputRootFrequency · l2BlockTime 등 프리셋 기본값 적용
trh-sdk L1 컨트랙트 배포
OptimismPortal · L1CrossDomainMessenger · L1StandardBridge · L2OutputOracle · SystemConfig
trh-sdk L2 Genesis 생성 + 체인 기동
GenesisPredeploys 삽입 (프리셋별 상이) → Sequencer · Batcher · Proposer 시작 → Helm 배포
trh-sdk 프리셋 모듈 자동 설치
PresetModules 맵 기준 — bridge · blockExplorer · monitoring · crossTrade · uptimeService (uptimeService는 자동, 나머지는 배포 후 Integration 탭에서 설정)
Backend 배포 결과 저장 + 상태 응답
deploymentId, stackId, rollupRpcUrl, explorerUrl, bridgeUrl, crossTradeUrl 반환
