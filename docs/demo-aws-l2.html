Feature 03
Stage A / B 분리 + 병렬 배포 아키텍처

  • EFS, NLB, ALB
  • Helm 차트 초기 설치
  • DNS / TLS 구성

  • DGF / OptimismPortal
  • DelayedWETH 등 초기화

→
errgroup
Stage B — L1 주소 필요
체인 노드 기동
  • 컨트랙트 주소 주입
  • Genesis 생성 + 동기화
  • op-geth / op-node 기동
  • Anchor State 제출
Stage A + L1 컨트랙트 동시 실행 · Stage B는 양쪽 완료 후 순차 진행 ~6분 단축
병렬화 효과
L1 init 함수가 L2 RPC에 의존하지 않음 — 코드 레벨 검증. Nonce 충돌 없음. errgroup 동시 실행으로 약 6분 단축.
분리의 또 다른 효과
실패 시 Stage B만 재시작 가능. EKS 프로비저닝(Stage A) 재실행 불필요. 디버깅 비용 대폭 감소.
