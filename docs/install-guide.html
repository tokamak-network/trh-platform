TRH Desktop 설치 및 실행

Tokamak Rollup Hub Desktop 앱으로 클릭 몇 번만에 나만의 L2 롤업을 로컬에서 배포하세요. Docker, 서버 설정 지식이 없어도 괜찮습니다.

macOS 12+ 지원
Docker 자동 설치
설치 시간 약 10분
1

TRH Desktop이란?

TRH Desktop은 Tokamak Rollup Hub 플랫폼을 비기술적 사용자도 손쉽게 실행할 수 있도록 만든 Electron 기반 데스크탑 앱입니다.

앱을 실행하면 내부적으로 Docker 컨테이너를 자동 관리하고, 설치 완료 후 브라우저 없이 앱 내부에서 바로 L2 대시보드를 사용할 수 있습니다.

💡
핵심 특징
Docker Desktop이 설치되지 않은 경우 앱이 자동으로 다운로드·설치합니다. 별도로 설치할 필요가 없습니다.

내부 구조

컴포넌트 역할 이미지
PostgreSQL 15 데이터베이스 (L2 상태 저장) postgres:15-alpine
trh-backend Node.js API 서버 tokamaknetwork/trh-backend:latest
trh-platform-ui Next.js 웹 대시보드 tokamaknetwork/trh-platform-ui:latest

앱 설치

1
DMG 파일 다운로드
Tokamak Network 공식 페이지 또는 GitHub Releases에서 TRH-Desktop-arm64.dmg (Apple Silicon) 또는 TRH-Desktop-x64.dmg (Intel Mac)을 다운로드합니다.
2
DMG 열기 및 앱 설치
다운로드한 .dmg 파일을 더블클릭하여 엽니다. 마운트된 창에서 TRH Desktop.app을 /Applications 폴더로 드래그합니다.
3
처음 실행 시 보안 허용
macOS Gatekeeper로 인해 "개발자를 확인할 수 없음" 경고가 표시될 수 있습니다. 아래 방법으로 실행하세요.
✅
해결 방법
방법 1: Finder에서 앱 우클릭 → 열기 → 열기 클릭
방법 2: 시스템 설정 → 개인 정보 보호 및 보안 → 하단 "그래도 열기" 클릭
4
앱 실행
Launchpad 또는 Applications 폴더에서 TRH Desktop을 클릭하여 실행합니다. 앱이 열리면 설정 화면이 표시됩니다.
4

1단계: 관리자 계정 설정

앱 실행 직후 관리자 계정 설정 화면이 표시됩니다. 두 가지 옵션 중 하나를 선택하세요.

옵션 A: 기본 계정 사용 (권장 — 빠른 시작)

사전 설정된 기본 계정으로 즉시 시작합니다. 개발·테스트 목적에 적합합니다.

항목값
Emailadmin@gmail.com
Passwordadmin123
⚠️
보안 주의
기본 계정은 외부에 공개된 환경에서 사용하지 마세요. 프로덕션 배포 시에는 반드시 커스텀 계정을 설정하세요.

옵션 B: 커스텀 계정 설정

직접 이메일과 비밀번호를 입력합니다.

📝
입력 규칙
이메일: 유효한 이메일 형식 (예: you@example.com)
비밀번호: 최소 8자 이상

설정 완료 후 Continue 버튼을 클릭하면 자동 설치 단계로 진행됩니다.

6

2단계: 자동 설치

설정 완료 후 앱이 자동으로 환경을 구성합니다. 5가지 단계를 순서대로 진행하며, 완료까지 약 5~10분이 소요됩니다.

🐳
Step 1 — Docker 환경 확인
Docker Desktop이 설치되어 있는지 확인합니다. 미설치 시 Install Docker 버튼이 표시되고, 클릭 시 자동으로 .dmg를 다운로드하여 설치합니다 (macOS 전용).
ℹ️
자동 설치 과정
Docker DMG 다운로드 (~600MB) → 마운트 → /Applications에 복사 → Docker.app 실행 → 재시도
📦
Step 2 — 컨테이너 이미지 다운로드
Docker Hub에서 PostgreSQL, Backend, Platform UI 이미지를 다운로드합니다. 처음 실행 시 네트워크 속도에 따라 3~8분 소요될 수 있습니다. 진행률 표시줄로 확인할 수 있습니다.
▶️
Step 3 — 서비스 시작
3개의 Docker 컨테이너(postgres, backend, platform-ui)를 순서대로 시작합니다. 포트 충돌이 감지되면 모달로 안내하며 자동 해제할 수 있습니다.
🔧
Step 4 — 백엔드 의존성 설치
컨테이너 내부에 필요한 도구(pnpm, node, forge, aws-cli)를 설치합니다. 이미 설치된 경우 이 단계는 빠르게 완료됩니다.
✅
Step 5 — 헬스 체크
모든 서비스가 정상 응답하는지 확인합니다. 완료되면 자동으로 Ready 화면으로 이동합니다.
⏱️
시간이 오래 걸린다면?
인터넷 속도에 따라 이미지 다운로드가 느릴 수 있습니다. 하단 터미널 패널에서 실시간 로그를 확인할 수 있습니다. 오류 발생 시 Retry 버튼이 표시됩니다.
7

3단계: 대시보드 사용

설치가 완료되면 소행성 애니메이션이 있는 Ready 화면이 표시됩니다.

대시보드 열기

Get Started 또는 Dashboard 버튼을 클릭하면 앱 내부에서 L2 관리 대시보드(localhost:3000)가 열립니다.

🖥️
인앱 브라우저
대시보드는 앱 내부 WebContentsView로 표시됩니다. 상단 네비게이션 바에서 뒤로/앞으로/새로고침 버튼과 ← Dashboard 버튼으로 메인 화면으로 돌아올 수 있습니다.

업데이트 알림

새로운 플랫폼 버전이 있으면 Ready 화면에 업데이트 배너가 표시됩니다. Update Now를 클릭하면 자동으로 최신 이미지를 받아 재시작합니다.

앱 종료

앱 창을 닫으면 macOS 트레이(상단 메뉴바)에 TRH 아이콘으로 남아 백그라운드에서 계속 실행됩니다. 완전히 종료하려면 트레이 아이콘 → Quit을 선택하세요. 종료 시 Docker 컨테이너가 자동으로 중지됩니다.

🔄
다음 실행부터는 빠릅니다
처음 설치 후에는 이미지가 로컬에 캐시되므로, 다음 실행 시에는 컨테이너가 즉시 시작됩니다 (약 30초).
8

포트 정보

TRH Desktop은 아래 포트를 사용합니다. 해당 포트를 사용하는 다른 서비스가 있으면 충돌이 발생할 수 있습니다.

포트 서비스 설명 접근 방법
3000 Platform UI Next.js 웹 대시보드 앱 내 또는 localhost:3000
8000 Backend API Node.js REST API localhost:8000
5433 PostgreSQL 데이터베이스 localhost:5433 (postgres/postgres)
ℹ️
PostgreSQL 포트가 5433인 이유
로컬에 이미 PostgreSQL(5432)이 설치된 경우 충돌을 방지하기 위해 5433을 사용합니다. 컨테이너 내부에서는 5432를 사용합니다.
9

문제 해결

자주 발생하는 문제와 해결 방법입니다.

🐳 Docker 자동 설치가 실패합니다 ▼

자동 설치 실패 시 수동으로 설치할 수 있습니다.

  • docker.com에서 Docker Desktop을 다운로드합니다
  • Apple Silicon Mac: Docker-arm64.dmg 선택
  • Intel Mac: Docker-x86_64.dmg 선택
  • 설치 완료 후 Docker Desktop을 한 번 실행하여 초기화 완료
  • TRH Desktop에서 Retry 버튼 클릭
🔴 포트 충돌 오류가 발생합니다 ▼

앱이 포트 충돌을 감지하면 모달로 안내합니다. Confirm을 클릭하면 자동으로 해당 프로세스를 종료합니다.

수동으로 해제하려면:

# 3000 포트를 사용 중인 프로세스 확인 lsof -i :3000 # PID로 종료 kill -9 <PID>
⏱️ 이미지 다운로드가 너무 오래 걸립니다 ▼

처음 설치 시 약 300~500MB의 Docker 이미지를 다운로드합니다. 인터넷 속도에 따라 다르며 보통 3~10분 소요됩니다.

  • 와이파이 대신 유선 연결을 사용하면 더 빠릅니다
  • 회사 VPN이 연결된 경우 속도가 느릴 수 있으니 VPN 해제 후 시도하세요
  • 오류 발생 시 Retry를 클릭하면 이어받기를 시도합니다
❌ 헬스 체크에서 Timeout이 발생합니다 ▼

컨테이너가 시작됐지만 서비스가 정상 응답하지 않는 경우입니다.

  • Retry 버튼 클릭 — 대부분의 경우 해결됩니다
  • Docker Desktop을 재시작한 뒤 앱을 다시 실행하세요
  • 메모리 부족 시 다른 앱을 닫고 시도하세요 (8GB+ 권장)

직접 컨테이너 상태를 확인하려면:

docker ps docker logs trh-backend --tail 50
🔐 macOS "개발자를 확인할 수 없음" 경고 ▼

앱이 Apple 공증(Notarization)을 받지 않은 경우 표시됩니다.

  • Finder에서 앱 아이콘을 우클릭 → 열기 선택
  • "개발자를 확인할 수 없음" 경고에서 열기 클릭
  • 이후 실행부터는 경고 없이 열립니다
💾 "디스크 공간 부족" 오류 ▼

Docker 이미지 다운로드 실패 시 자동으로 Docker 디스크 정리를 시도합니다. 수동으로 정리하려면:

# 사용하지 않는 Docker 데이터 정리 docker system prune -af docker volume prune -f

⚠️ 위 명령은 다른 Docker 컨테이너 데이터도 삭제될 수 있으니 주의하세요.

10

자주 묻는 질문

앱을 닫으면 L2 노드도 종료되나요? ▼

macOS에서는 창을 닫아도 앱이 메뉴바 트레이에 남아 백그라운드에서 계속 실행됩니다. Docker 컨테이너도 계속 동작합니다. 완전히 종료하려면 메뉴바 트레이 아이콘 → Quit을 선택하세요. 종료 시 컨테이너가 자동 중지됩니다.

데이터는 어디에 저장되나요? ▼

모든 데이터는 Docker 볼륨에 저장됩니다:

  • trh_postgres_data — 데이터베이스
  • trh_backend_data — 백엔드 데이터
  • trh_backend_logs — 로그

앱을 재설치하거나 새 버전으로 업데이트해도 데이터는 보존됩니다.

업데이트는 어떻게 하나요? ▼

새 버전이 있으면 Ready 화면에 업데이트 배너가 자동으로 표시됩니다. Update Now를 클릭하면:

  • 최신 Docker 이미지를 자동으로 Pull
  • 컨테이너 재시작
  • 헬스 체크 완료 후 새 버전 사용 가능
대시보드 기본 로그인 정보가 무엇인가요? ▼

설정 단계에서 기본 계정을 선택한 경우:

  • Email: admin@gmail.com
  • Password: admin123

커스텀 계정을 선택했다면 설정 단계에서 입력한 계정을 사용하세요.

완전히 삭제하려면 어떻게 하나요? ▼

앱과 데이터를 모두 제거하려면:

# 1. 컨테이너와 볼륨 삭제 docker compose down -v # (docker-compose.yml 파일이 있는 경로에서 실행) # 2. 앱 삭제 rm -rf /Applications/"TRH Desktop.app"
