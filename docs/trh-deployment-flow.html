Repository Dependency Map

각 레포지토리가 배포 과정에서 어떤 역할을 하고, 어떻게 연결되는지의 요약입니다.

● trh-platform (Electron)
  • Docker 컨테이너 라이프사이클 관리
  • trh-backend + trh-platform-ui 호스팅
  • Keystore / 자격증명 주입
  • 의존: Docker Engine
● trh-platform-ui (Next.js)
  • 사용자 대면 웹 대시보드
  • Preset 선택 위저드
  • Funding 상태 / 배포 진행률 표시
  • 의존: trh-backend API
● trh-backend (Go API)
  • REST API 서버 (port 8000)
  • Preset 정의 및 매개변수 변환
  • 계정 파생, 펀딩 상태 추적
  • 의존: trh-sdk (Go 라이브러리), PostgreSQL
● trh-sdk (Go SDK/CLI)
  • L1 컨트랙트 배포 실행 (Forge)
  • L2 인프라 배포 (Docker / Terraform)
  • Fee Token, Docker 이미지 태그 상수
  • 의존: tokamak-thanos (git clone)
● tokamak-thanos (OP Stack Fork)
  • Solidity 컨트랙트 소스 (Deploy.s.sol)
  • Genesis predeploy 정의
  • L2 컴포넌트 소스 (op-geth, op-node 등)
  • 역할: SDK가 런타임에 클론하여 사용
● tokamak-thanos-stack (IaC)
  • AWS Terraform 인프라 프로비저닝
  • Helm 차트 (Kubernetes)
  • Preset별 values 파일
  • 역할: AWS 배포 시에만 사용 (Local은 미사용)
